引言#

斯德哥尔摩作为北欧最大的数据中心枢纽之一，凭借极低的自然冷却成本、稳定的电力供应以及严格的数据隐私法规，成为众多云服务商的重要部署节点。这座城市不仅是连接北欧与欧洲大陆的网络交汇点，还能够覆盖整个斯堪的纳维亚半岛及波罗的海。

JustHost 在斯德哥尔摩部署的机房正是看中了这些天然优势，为需要欧洲节点的用户提供了一个性价比选择。本次测评针对 JustHost 瑞典斯德哥尔摩机房的 Sirius 基础套餐展开，通过网络延迟、带宽实测、磁盘性能、CPU 跑分等多个维度的数据。

性能实测数据#

通过一系列标准化测试工具，对这台斯德哥尔摩机房 VPS 的 CPU、内存、磁盘和网络性能进行全面评估。

CPU 单核性能#

使用 sysbench 进行 CPU 运算能力测试，限制素数计算上限为 10000:

root@r1089906:~# sysbench cpu --threads=1 run
CPU speed:
    events per second:   894.58
Latency (ms):
         min:                                    1.04
         avg:                                    1.12
         95th percentile:                        1.25

单核心每秒完成 894.58 次事件处理，平均延迟 1.12 毫秒。这个成绩在至强 Gold 6252 处理器上属于正常水平，毕竟基础频率只有 2.1GHz，主要依靠架构优势而非频率。对于日常建站、运行脚本或轻量级应用来说，这个性能完全够用。

内存读写速度#

root@r1089906:~# sysbench memory run
Total operations: 41919606 (4190966.06 per second)
40937.12 MiB transferred (4092.74 MiB/sec)
Latency (ms):
         avg:                                    0.00
         95th percentile:                        0.00

内存写入速度达到 4092.74 MiB/sec，这是 DDR4 服务器内存的标准表现。由于测试的是虚拟化环境，实际速度会受到宿主机内存分配策略的影响，但从延迟几乎为零来看，内存调度效率不错，不会成为系统性能瓶颈。

磁盘 I/O 性能#

4K 随机读写测试 是衡量 NVMe 固态硬盘真实性能的关键指标:

root@r1089906:~# fio --name=randrw --rw=randrw --bs=4k
  read: IOPS=10.1k， BW=39.5MiB/s
    clat (usec): avg=83.61
  write: IOPS=10.1k， BW=39.5MiB/s
    clat (nsec): avg=6301.87

4K 随机读取 IOPS 达到 10100，带宽 39.5 MiB/s，读取延迟平均 83.61 微秒;随机写入同样是 10100 IOPS，写入延迟仅 6.3 微秒。这个数据确实符合 NVMe 固态硬盘的特征，尤其是微秒级的延迟表现，证明 JustHost 没有虚标，确实使用了 NVMe 存储。

顺序读写测试结果更加亮眼:

root@r1089906:~# fio --name=seqrw --rw=readwrite --bs=1M
  read: IOPS=746， BW=747MiB/s
  write: IOPS=795， BW=795MiB/s

1MB 块大小的顺序读取速度 747 MiB/s，顺序写入 795 MiB/s，这已经接近中端 NVMe 硬盘的理论性能。对于数据库查询、日志写入、网站静态资源加载等场景，这样的磁盘速度能够显著减少 I/O 等待时间，提升整体响应速度。

从整体性能来看，这台 VPS 的硬件配置货真价实，NVMe 硬盘的 I/O 性能尤其突出，适合对磁盘读写有要求的场景。CPU 和内存虽然配置较低，但对于中小型网站、开发测试环境或轻量级应用完全胜任。

中国访问网络表现#

对于需要从国内访问或管理这台瑞典 VPS 的用户，网络延迟和稳定性是重要考量因素。通过多项测试评估从中国大陆访问斯德哥尔摩机房的实际体验。

回国延迟测试#

从 VPS 向国内主流网站发起 ping 测试:

root@r1089906:~# ping -c 4 www.baidu.com
PING www.wshifen.com (103.235.46.102) 56(84) bytes of data.
64 bytes from 103.235.46.102: icmp_seq=1 ttl=34 time=268 ms
64 bytes from 103.235.46.102: icmp_seq=2 ttl=34 time=268 ms
--- www.wshifen.com ping statistics ---
4 packets transmitted， 4 received， 0% packet loss， time 3001ms
rtt min/avg/max/mdev = 267.667/267.772/267.842/0.066 ms

到百度服务器的平均延迟稳定在 268 毫秒左右，波动极小(标准差仅 0.066ms)，且无丢包。这个延迟符合中欧线路的正常水平，虽然不适合实时游戏或视频通话，但用于 SSH 远程管理、网站后台操作基本感觉不到明显卡顿。

丢包率稳定性#

24 小时监测显示整体丢包率为 0.21%，基本可以忽略不计。但在晚高峰时段(19:00-23:00)丢包率会上升至 6% 左右，这是因为国际带宽在用网高峰期出现拥堵。

路由追踪路径#

从中国上海出发到斯德哥尔摩的数据包路径为:上海 → 荷兰阿姆斯特丹 → 瑞典斯德哥尔摩。这条线路经过欧洲骨干网中转，没有绕道亚洲其他国家，路由相对直接。阿姆斯特丹作为欧洲最大的互联网交换中心之一，在此转接能够保证较好的带宽质量。

10 秒测试中平均下载速度达到 66 Mbps，峰值接近 98 Mbps，已经占用了家庭宽带 100Mbps 带宽的 60% 以上。考虑到跨越半个地球的距离和国际线路损耗，这个速度相当不错。

启用 BBR 算法后明显改善了 TCP 拥塞控制，前 2 秒建立连接后速度迅速爬升，中途虽有波动但整体稳定在 60-90 Mbps 区间。

补充测试项目#

IP 质量检测#

通过 IP 质量检测脚本对这台 VPS 的 IP 地址进行综合评估，结果显示 IP 归属于 AS26383 ASNET，注册地为阿联酋但实际使用地在瑞典。多个数据库均识别为机房 IP，这是 VPS 的正常属性。

风险评分方面，IP2Location、Scamalytics、ipapi 和 AbuseIPDB 均给出低风险或极低风险评级，仅 IPQS 标记为 75 分的可疑 IP。重要的是 IP 未出现在任何黑名单数据库中(439 个数据库检测，0 个黑名单)，说明这个 IP 段比较干净。

邮件端口方面，25 端口出站可用，且能够正常连接 Gmail、Outlook、Yahoo 等主流邮件服务商。

流媒体解锁情况#

流媒体解锁测试结果比较常规:

  • 已解锁:Netflix(瑞典区)、YouTube Premium(美区)、Amazon Prime Video(德区)、iQyi 海外版、ChatGPT、 Gemini、Claude

Netflix 能够解锁瑞典本地内容，YouTube Premium 识别为美国区域。Disney+ 被 IP 封禁属于意料之中，流媒体平台对机房 IP 的检测越来越严格。值得一提的是主流 AI 服务(ChatGPT、Gemini、Claude)均可正常访问。

整体来看，这个 IP 质量中规中矩，没有黑历史记录，部分流媒体可用，适合建站、开发测试等常规用途。
