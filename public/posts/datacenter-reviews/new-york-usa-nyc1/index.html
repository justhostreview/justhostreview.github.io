NYC1 机房的性能实测数据#

通过一系列标准化测试工具，对 JustHost 纽约 NYC1 机房的 Sirius 套餐进行全面性能评估，以下是各项测试的详细结果。

CPU 运算能力#

使用 sysbench 进行 CPU 基准测试，计算质数的运算速度：

root@r1089906:~# sysbench cpu --threads=1 run
Prime numbers limit: 10000
CPU speed:
    events per second:   599.58
Latency (ms):
         min:                                    1.28
         avg:                                    1.66
         95th percentile:                        2.11

单核心每秒完成 599.58 次事件运算，平均延迟 1.66 毫秒。将测试难度提升到 20000 质数上限后，每秒事件数降至 245.65，这个成绩对于 E5-2683 v4 这样的老架构处理器来说属于正常水平。

应对建站、Python 脚本、小型数据库等日常任务完全够用，但不适合高并发计算或视频转码等 CPU 密集型场景。

磁盘 I/O 性能#

4K 随机读写测试（模拟数据库、网站等实际应用场景）：

randrw: (groupid=0, jobs=1): err= 0
  read: IOPS=4324, BW=16.9MiB/s
    clat (usec): avg=190.36, 95th percentile: 310
  write: IOPS=4330, BW=16.9MiB/s
    clat (usec): avg=22.73, 95th percentile: 37

4K 随机读取 IOPS 达到 4324，写入 IOPS 为 4330，这个成绩确实符合 NVMe 固态硬盘的性能特征。

传统 SATA SSD 的 4K 随机读写 IOPS 通常在 1000-2000 左右，而这款 VPS 的表现明显高出一个档次。读取平均延迟 190 微秒、写入延迟仅 22 微秒，响应速度非常快，对数据库查询、网站动态内容加载都有积极作用。

顺序读写测试（大文件传输场景）：

seqrw: (groupid=0, jobs=1): err= 0
  read: IOPS=581, BW=581MiB/s (610MB/s)
  write: IOPS=618, BW=619MiB/s (649MB/s)

顺序读取速度 581 MB/s，写入速度 619 MB/s。虽然没有达到高端 NVMe 硬盘 3000+ MB/s 的顶级水平，但这个速度已经远超普通 SATA SSD 的 500 MB/s 上限，证明 JustHost 使用的确实是 NVMe 协议的固态硬盘，而非虚标。

回国网络表现测试#

对于国内用户来说，VPS 访问国内网络的延迟和稳定性至关重要。以下针对 JustHost 纽约 NYC1 机房到中国大陆的网络连接进行了多维度测试。

全国多地延迟分布#

使用 itDog 平台测试从中国各地到 VPS 的延迟情况：

从数据可以看出，电信线路表现最好，平均延迟 232ms，沿海发达地区延迟普遍在 200ms 左右；联通居中，平均 256ms；移动稍逊，平均 269ms。香港节点延迟 194ms 是全国最低点，西藏拉萨等偏远地区延迟突破 300ms。

总体来说，这个延迟水平适合建站、API 服务等对实时性要求不太苛刻的应用，但不太适合需要快速响应的场景。

网络丢包率测试#

全国综合丢包率为 8.85%，这个数值相对偏高。丢包率接近 9% 意味着每 100 个数据包就有约 9 个会丢失或需要重传，对网页加载速度、文件传输稳定性都会产生一定影响。这可能与跨太平洋海底光缆的拥堵情况、运营商国际出口带宽限制等因素有关。

如果对网络稳定性要求较高，建议考虑使用 CN2 GIA 等优化线路的商家（例如 搬瓦工 ），或者选择亚太地区的节点。

路由追踪分析#

从中国上海出发的数据包，经过美国加利福尼亚州圣何塞，最终到达美国纽约州布法罗。这条路由路径显示数据包需要先从上海跨越太平洋到达美国西海岸，再横穿整个美国大陆到达东海岸纽约，没有明显的绕路问题，延迟较高也在合理范围内。

Q2: JustHost 的 fair-share 带宽是什么意思？#

+
Fair-share（公平共享）带宽模式是指用户可以使用最高到标称值的带宽（如 300Mbps），但在网络繁忙时段，带宽会根据实际负载情况在多个用户之间动态分配。这种模式的优点是非高峰时段可以获得更高带宽，缺点是无法保证持续的独享带宽。

Q3: JustHost VPS 适合建站吗？#

+
适合。Sirius 这类基础配置足以运行 WordPress、Typecho 等常见建站程序，NVMe 硬盘的高 IOPS 对数据库查询有积极作用。JustHost 默认没有配置 Swap，如果选择小内存 VPS，建议手动配置 Swap 交换空间。

Q4: JustHost 回国网络延迟 200+ 毫秒怎么样？#
