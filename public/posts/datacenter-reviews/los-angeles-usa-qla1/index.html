内存读写速度#

内存测试跑出了 2843.02 MiB/sec 的写入速度，每秒完成近 291 万次操作。这个成绩说明 DDR4 内存性能发挥正常，不存在明显的虚拟化性能损耗。对于需要频繁内存读写的应用来说，这个速度足够流畅。

磁盘 IO 性能表现#

4K 随机读写测试：

使用 fio 进行 4K 随机读写测试，读取 IOPS 达到 3573，写入 IOPS 同样为 3573，读写带宽均为 14.0 MiB/s。这个成绩对于 NVMe 固态硬盘来说属于中等偏下水平——真正的高性能 NVMe 硬盘 4K 随机读写 IOPS 通常能达到几万甚至十几万。

read: IOPS=3573, BW=14.0MiB/s (14.6MB/s)
write: IOPS=3573, BW=14.0MiB/s (14.6MB/s)

从数据来看，JustHost 标称的"NVMe"可能存在一定的虚拟化限制或者共享存储架构，并非单独分配的高性能 NVMe 设备。不过这个性能应对建站、数据库读写等常规场景依然够用，只是不适合对磁盘 IO 要求极高的应用。

顺序读写测试：

大文件顺序读写测试中，读取速度 454 MiB/s，写入速度 483 MiB/s，这个成绩就比较符合 NVMe 的表现了。顺序读写性能不错，适合处理大文件传输、备份等操作。

READ: bw=454MiB/s (476MB/s)
WRITE: bw=483MiB/s (507MB/s)

综合来看，这块硬盘在小文件随机读写上受到一定限制，但大文件顺序读写能力正常，属于虚拟化环境下的典型表现。

丢包率测试#

通过 itDog 进行多节点丢包率测试，整体丢包率约为 5%。这个丢包率稍微偏高，在网络高峰期可能会影响连接稳定性。如果是跑网站或者 API 服务问题不大，但用于实时通讯、游戏加速等场景就不太合适了。

路由追踪分析#

使用 ipip.net 从上海电信节点追踪到洛杉矶机房的路由路径，数据包从中国上海出发，经过美国加利福尼亚州圣何塞的骨干网节点，最终到达洛杉矶目标服务器。

从路由路径来看，JustHost 洛杉矶机房走的是常规国际线路，没有接入 CN2 GIA 等优化线路。这也解释了为什么延迟在 160ms 左右且存在一定丢包率——普通线路在晚高峰时段容易出现拥堵。

实际下载带宽测试#

从国内家庭 100Mbps 电信宽带环境，使用 iperf3 测试从 VPS 下载文件的实际速度。测试结果显示，稳定传输速度维持在 118-120 Mbits/sec 左右，平均接收速度 101 Mbits/sec，几乎跑满了本地带宽。

这个成绩非常不错，说明虽然延迟和丢包率不算完美，但带宽利用率很高。下载大文件、同步数据等操作速度令人满意，重传次数为 0 也证明了连接质量相对稳定。

IP 质量与流媒体解锁#

IP 质量检测#

通过 IPQuality 脚本对服务器 IP 进行全面检测，这个 IP 地址被识别为美国洛杉矶的原生 IP，归属于 AS26383 ASNET。从多个数据库的判定来看，IP 类型均被标记为"机房"性质。

风险评分方面表现一般：IP2Location 和 AbuseIPDB 评定为低风险，但 Scamalytics 给出了 74 分的高风险评级，IPQS 也将其标记为"可疑 IP"。这种情况在 VPS IP 中比较常见，主要是因为机房 IP 段被多人使用过，可能存在历史不良记录。

值得关注的是邮件发送能力：25 端口出站可用，但部分邮局无法正常通信。
