基础系统信息#

在开始性能测试之前，先了解一下这台圣彼得堡机房 VPS 的硬件配置和系统环境。通过系统命令可以获取处理器型号、内存分配、磁盘使用等基础信息，这些参数直接影响后续的性能表现。

Ubuntu 22.04.5 LTS

系统运行的是 Ubuntu 22.04.5 LTS 长期支持版本，内核版本为 5.15.0-122。这个内核版本比较新，对 NVMe 固态硬盘的支持和网络栈优化都比较到位，适合跑各类应用服务。

CPU 配置#

root@r1089906:~# lscpu | egrep 'Model name|Socket|Core|Thread|MHz'

Thread(s) per core:                   1

分配的处理器是 Intel Xeon E5-2699 v3，这是 Haswell 架构的至强处理器，主频 2.30GHz。虽然是老一代服务器 CPU，但 E5-2699 v3 属于当年的旗舰型号，单核性能依然够用。

内存状态#

root@r1089906:~# free -h
               total        used        free      shared  buff/cache   available
Mem:           957Mi       200Mi       157Mi       1.0Mi       599Mi       600Mi
Swap:             0B          0B          0B

实际可用内存为 957MB，接近标称的 1GB(扣除系统保留部分后的正常值)。需要注意的是，JustHost 默认没有配置 Swap 交换分区，这在小内存 VPS 上可能是个隐患。当物理内存耗尽时，系统会直接 OOM 杀进程，而不是使用磁盘作为虚拟内存缓冲。

建议运行内存占用较高应用的用户手动创建 Swap 文件，一般设置 1-2GB 即可，避免内存不足导致服务异常。

圣彼得堡性能实测#

通过一系列基准测试，可以全面了解这台圣彼得堡 VPS 在 CPU 运算、内存读写、磁盘 I/O 以及网络带宽方面的真实表现。以下测试均在系统空载状态下进行，结果具有参考价值。

CPU 性能#

使用 Sysbench 进行 CPU 单核计算能力测试:

root@r1089906:~# sysbench cpu --threads=1 run
CPU speed:
    events per second:   802.14

Latency (ms):
         min:                                    1.10
         avg:                                    1.24
         95th percentile:                        1.52

单核每秒处理 802 个事件，平均延迟 1.24 毫秒。这个成绩对于 E5-2699 v3 来说属于正常水平，虽然是 2014 年的处理器，但应付网站、数据库等常规应用没有压力。需要注意的是只有 1 个 vCore，多线程应用会受到限制，适合单进程或轻量服务。

内存性能#

root@r1089906:~# sysbench memory run
Total operations: 39215825 (3919998.90 per second)
38296.70 MiB transferred (3828.12 MiB/sec)

内存写入速度达到 3828 MB/s，这个数字符合 DDR4 内存的表现。对于 VPS 环境来说，内存性能主要取决于宿主机配置，圣彼得堡机房在这方面没有明显短板，日常应用不会遇到内存瓶颈。

磁盘 I/O 性能#

4K 随机读写测试:

root@r1089906:~# fio --name=randrw --rw=randrw --bs=4k --size=512M
  read: IOPS=7203， BW=28.1MiB/s (29.5MB/s)
    clat (usec): avg=113.94
  
  write: IOPS=7195， BW=28.1MiB/s (29.5MB/s)
    clat (usec): avg=14.04

4K 随机读写 IOPS 均在 7200 左右，带宽约 28 MB/s。读取平均延迟 114 微秒，写入仅 14 微秒。这个成绩基本达到了 NVMe SSD 的入门水平，虽然不如高端 NVMe 能跑到几万 IOPS，但对比传统 SATA SSD 已经有明显优势。

顺序读写测试:

root@r1089906:~# fio --name=seqrw --rw=readwrite --bs=1M --size=1G
  read: IOPS=757， BW=757MiB/s (794MB/s)
  write: IOPS=806， BW=806MiB/s (845MB/s)

1MB 块大小的顺序读取速度 757 MB/s，写入速度 806 MB/s。这个成绩非常不错，接近高性能 NVMe 的理论值。对于需要频繁读写大文件的场景，比如备份恢复、日志处理等，这样的磁盘性能完全够用。

回国网络表现#

对于国内用户来说，最关心的还是从圣彼得堡机房访问国内网络的实际质量。通过多个维度的测试，可以了解这条俄罗斯线路在延迟、丢包、路由以及实际下载速度方面的真实表现。

首先通过 ping 百度来测试基础延迟:

root@r1089906:~# ping -c 4 www.baidu.com
PING www.wshifen.com (103.235.46.115) 56(84) bytes of data.

64 bytes from 103.235.46.115: icmp_seq=2 ttl=34 time=283 ms
64 bytes from 103.235.46.115: icmp_seq=3 ttl=34 time=283 ms
64 bytes from 103.235.46.115: icmp_seq=4 ttl=34 time=283 ms

rtt min/avg/max/mdev = 282.821/282.869/282.992/0.071 ms

延迟稳定在 283 毫秒左右，抖动极小(标准差仅 0.071ms)，四次测试无丢包。这个延迟数字对于俄罗斯到中国的物理距离来说属于正常水平，但对于需要实时交互的应用(比如游戏、视频会议)会有明显的卡顿感。

全国多节点延迟分布#

使用 itDog 工具测试国内各省份节点的延迟表现:

从测试数据看，全国平均延迟 257ms，其中:

  • 电信线路:天津最快 193ms，陕西延安最慢 299ms，平均 234ms
  • 联通线路:北京最快 138ms，广东广州 317ms，平均 229ms
  • 移动线路:北京 253ms，西藏拉萨最慢 378ms，平均 302ms

可以看出联通线路表现最好，北京联通节点仅 138ms，这说明联通与俄罗斯的互联质量相对较优。移动线路整体延迟偏高，尤其是西藏等偏远地区超过 370ms。电信居中，但波动范围也比较大。

丢包率测试#

全国节点平均丢包率 3.85%，这个数字已经超过了理想状态(通常认为 1% 以下为良好)。接近 4% 的丢包率会对 TCP 连接质量产生影响，表现为网页加载缓慢、文件传输速度不稳定等问题。

从实际使用角度看，这条线路不适合对稳定性要求高的生产环境，更适合测试、备份或对丢包不敏感的应用。

路由追踪#

从中国上海出发的数据包，先进入德国黑森州法兰克福，然后经过丹麦和瑞典，最终到达俄罗斯圣彼得堡。这条路径绕了一大圈，并非直连俄罗斯，而是通过欧洲中转。这解释了为什么延迟会达到 280ms 以上——地理距离加上多次跳转导致延迟累积。

10 秒内传输 94.2MB，平均速度 79 Mbps，基本跑满了百兆宽带的下行容量。测试过程中速度从初始的 2.75 Mbps 快速爬升到 90+ Mbps，说明 TCP 拥塞控制算法运行正常，能够充分利用带宽。重传次数为 0，说明在测试时段链路质量稳定。

需要注意的是，这个测试是在白天非高峰时段进行的。晚上 8-11 点国内网络高峰期，实际速度可能会有所下降。另外测试用的是电信宽带，前面延迟测试显示联通线路更优，如果使用联通宽带下载速度可能会更好。

补充测试#

除了基础性能和网络延迟，IP 质量和流媒体解锁能力也是评估 VPS 实用性的重要指标。以下是针对圣彼得堡机房的补充测试。

IP 质量检测#

通过多个数据库交叉验证，该 IP 地址被所有平台识别为机房 IP，这对于 VPS 来说是正常情况。风险评分方面，大部分数据库给出了低风险评级，但 IPQS 将其标记为 “可疑 IP”(评分 75)，ipapi 显示 5.08% 的高风险率。

IP 类型明确标注为广播 IP 和服务器用途，未被标记为 VPN 或代理，这意味着在大多数场景下不会被当作代理 IP 封禁。邮局连通性测试显示本地 25 端口可用，但在 439 个黑名单数据库中有 3 个将该 IP 列入黑名单，这对发送邮件可能有轻微影响。

流媒体解锁测试#

流媒体解锁能力比较有限:

  • 不可用: Disney+、Netflix、YouTube Premium、Amazon Prime Video、Spotify、Reddit 等主流平台均无法访问
  • 部分可用: TVBAnywhere+ 可以正常使用，ChatGPT 仅支持移动 APP 访问
  • 区域限制: Apple、Google Play Store、Steam 等服务识别为俄罗斯区域

Netflix 甚至直接显示 IP 被封禁，说明该机房的 IP 段已被流媒体平台列入黑名单。如果需要解锁流媒体服务，建议选择专门的原生 IP 机房或者其他对流媒体友好的地区节点。圣彼得堡机房更适合纯粹的技术用途，而非娱乐或内容消费场景。
