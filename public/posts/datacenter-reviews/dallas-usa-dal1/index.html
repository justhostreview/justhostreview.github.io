达拉斯位于美国德克萨斯州北部，是美国重要的电信枢纽城市之一。这里汇集了多条国际海底光缆和骨干网络，网络基础设施成熟，到美国东西海岸的延迟都比较均衡。相比洛杉矶、圣何塞等西海岸热门节点，达拉斯机房的带宽资源更丰富。

JustHost 在达拉斯设有数据中心，主要面向需要美国中南部节点的用户。这个机房到国内三大运营商的路由表现如何？实际速度能跑满吗？延迟是否稳定？本文通过实际测试数据，对 JustHost 达拉斯机房的网络性能进行详细评测。

磁盘 IO 性能#

4K 随机读写测试

这项测试更接近真实使用场景，特别是数据库和网站程序的读写模式。测试结果显示随机读取 IOPS 为 6934，写入 IOPS 为 6926，读写带宽均为 27.1 MiB/s。

read: IOPS=6934, BW=27.1MiB/s (28.4MB/s)
    clat (usec): avg=120.70, stdev=1207.65
    
write: IOPS=6926, BW=27.1MiB/s (28.4MB/s)
    clat (usec): avg=14.10, stdev=46.54

随机读延迟平均 120 微秒，写延迟仅 14 微秒，写入速度明显快于读取。IOPS 接近 7000 的表现对于 NVMe 硬盘来说属于中等偏上水平，运行 MySQL、PostgreSQL 等数据库不会出现明显的 IO 瓶颈。

顺序读写测试

大文件连续读写速度更能体现硬盘的峰值性能。测试显示顺序读取速度 741 MiB/s，写入速度 789 MiB/s，读取延迟 516 微秒，写入延迟 722 微秒。

read: IOPS=741, BW=741MiB/s (777MB/s)
write: IOPS=789, BW=789MiB/s (828MB/s)

顺序读写速度都突破了 700 MiB/s，这个成绩相当不错。备份数据、上传大文件或者进行日志写入时，能明显感受到 NVMe 固态硬盘的速度优势。

带宽速度测试#

使用 Speedtest 连接达拉斯本地的 Hivelocity 测速节点，测试结果如下：

Idle Latency:     0.61 ms   (jitter: 0.05ms)
    Download:   350.83 Mbps
      Upload:   411.26 Mbps
 Packet Loss:     0.0%

下载速度 350.83 Mbps，上传速度 411.26 Mbps，均超过了标称的 300 Mbit 带宽。本地延迟仅 0.61 毫秒，抖动极小，丢包率为 0。上传速度反而比下载快，这在 VPS 中比较少见，说明机房的上行带宽资源比较充足。

需要说明的是，这个 300 Mbit 带宽是 fair-share 共享模式，也就是说在非高峰时段可以跑满甚至超出标称带宽，但在机房整体负载较高时可能会受到限制。从测试结果看，达拉斯机房的带宽资源比较宽裕，实际使用体验应该不错。

接近 190 毫秒的延迟属于美国中南部机房到中国大陆的正常水平。相比洛杉矶等西海岸节点（通常 150-170ms），达拉斯的地理位置确实要远一些。

使用 itDog 工具在晚高峰 18:00 进行全国多节点测试，结果显示三网延迟情况如下：

电信表现最稳定，全国各地延迟都在 200 毫秒左右。联通出现了一个异常节点（济南 751ms），剔除这个节点后整体表现也不错。移动网络相对慢一些，但延迟波动不大。

需要注意的是丢包率测试结果为 5.21%，这个数值偏高了。

5% 以上的丢包率在日常使用中可能会感觉到卡顿，特别是实时通信或在线游戏场景。不过测试时段是晚高峰，网络拥堵程度较高，非高峰时段的丢包情况应该会好一些。

路由追踪#

从国内访问达拉斯机房的路由路径为：中国上海 → 美国加州圣何塞 → 美国的城市 → 美国德州达拉斯。

数据包先从上海出口到美国西海岸的圣何塞，然后经过美国内陆节点中转，最终到达达拉斯。这条路由比较合理，中间没有绕路到欧洲或其他地区的情况。不过经过的跳数较多，也是延迟相对偏高的原因之一。

实际下载速度#

使用家庭 100Mbps 电信宽带，通过 iperf3 测试从达拉斯 VPS 下载数据的速度。测试过程中速度波动明显，前几秒只有几百 Kbps，随后逐渐提升到 20-35 Mbps 区间。

10 秒测试期间平均下载速度 17.5 Mbits/sec（约 2.2 MB/s），重传次数 1368 次。速度波动和重传次数都说明跨境网络质量不太稳定，和前面丢包率测试的结果相互印证。如果主要面向国内用户提供服务，这个回国速度只能说勉强够用。

下载个小文件或访问网页问题不大，但传输大文件或做实时数据同步就会比较慢。

补充测试#

IP 质量检测#

使用 IPQuality 脚本对达拉斯机房的 IP 地址进行全面检测，结果显示这是一个原生 IP 地址。

IP 归属于 AS398343 BAXET-GROUP，地理位置定位在达拉斯 75270 邮编区域。多个数据库识别结果显示：

  • IP 类型：原生 IP，所有主流数据库（IPinfo、ipregistry、ipapi、IP2Location）均识别为数据中心 IP
  • 风险评分：IP2Location 和 AbuseIPDB 评分较低（低风险），仅 Scamalytics 评分 64 分（高风险）
  • 风险因子：IPQS 数据库标记为代理和 VPN，其他数据库未标记相关风险
  • 黑名单检测：439 个有效数据库中，387 个正常，47 个已标记，5 个黑名单

邮件服务方面，本地 25 端口可用，但部分邮局通信受限。Gmail、Yahoo、MailCOM 等可以正常发送，QQ、163、Sohu、Sina、AOL、GMX 等邮局被屏蔽。如果需要搭建邮件服务器，需要注意国内邮局的连通性问题。

流媒体解锁测试#

使用 RegionRestrictionCheck 脚本测试各大流媒体平台的访问情况。

测试结果显示达拉斯机房可以正常观看 Netflix、Disney+ 等主流流媒体内容，识别地区均为美国。YouTube CDN 和 Netflix CDN 都位于达拉斯本地，访问速度应该不错。ChatGPT、Claude、Google Gemini 等 AI 服务也都可以正常使用。

唯一遗憾的是 Spotify 注册功能被限制，如果需要注册 Spotify 账号可能需要其他地区的 IP。整体来说，这个 IP 的流媒体解锁能力还算不错，能满足大部分使用需求。

测评总结#

JustHost 达拉斯机房的整体表现属于中规中矩的水平。硬件配置方面，Intel Xeon E5-2697 v4 处理器虽然不是最新型号，但单核性能稳定可靠；1GB 内存对于轻量级应用足够使用；NVMe 固态硬盘的随机 IOPS 接近 7000，顺序读写速度超过 700 MiB/s。

网络方面，本地带宽测试能跑到 350-411 Mbps，超出了 300 Mbit 的标称值，说明机房带宽资源比较充裕。流媒体解锁能力不错，主流平台都能正常访问，ChatGPT 等 AI 服务也完全可用。IP 质量属于原生 IP，仅有一个数据库标记为高风险。

回国网络是达拉斯机房的短板。190 毫秒左右的延迟虽然在可接受范围内，但 5% 以上的丢包率和不太稳定的下载速度会影响使用体验。不适合为国内用户提供服务，但如果业务主要在美国本土或者对回国速度要求不高，性价比还是相当不错的。
