CPU 处理器配置#

JustHost 香港机房使用的是 Intel Xeon E5-2697A v4 处理器，这是一款面向服务器市场的至强系列 CPU。基准主频 2.60GHz，虽然是几年前的架构，但性能依然够用。分配给 VPS 的是单核心单线程配置，这在入门级套餐中很常见。

Model name:                           Intel(R) Xeon(R) CPU E5-2697A v4 @ 2.60GHz
Thread(s) per core:                   1
Core(s) per socket:                   1
Socket(s):                            1

单核配置应付日常的网站访问、API 调用基本没问题，但如果需要跑计算密集型任务，可能会感觉吃力。适合的应用场景包括个人博客、静态网站、轻量级数据库等。

内存容量状况#

实际可用内存为 957MB，略小于标称的 1024MB，这是正常现象，因为系统内核会占用一部分内存。需要特别注意的是，JustHost 默认没有配置 Swap 交换空间。

               total        used        free      shared  buff/cache   available
Mem:           957Mi       156Mi       186Mi       1.0Mi       614Mi       643Mi
Swap:             0B          0B          0B

对于1GB内存的小配置 VPS 来说，没有 Swap 可能会在内存不足时直接导致进程被杀掉。建议手动创建1GB左右的 Swap 分区作为缓冲，防止内存用满时系统崩溃。虽然 Swap 速度比物理内存慢，但关键时刻能救急。

磁盘存储空间#

系统使用单个分区挂载在根目录，文件系统是 ext4 格式，总容量20GB。全新系统已经使用了4.7GB，剩余可用空间15GB。

Filesystem     Type  Size  Used Avail Use% Mounted on
/dev/sda1      ext4   20G  4.7G   15G  25% /

20GB NVMe 固态硬盘空间对于轻量级应用够用，但如果需要存储大量图片、视频或者数据库文件，可能需要定期清理或者升级到更大容量的套餐。建议保留至少30%的空闲空间，避免磁盘写满影响系统性能。

不过需要明确的是，单核配置的计算能力有限。如果网站需要处理大量并发请求，或者运行 WordPress 这类动态 CMS，在高峰期可能会感觉响应变慢。适合访问量不大的个人博客、API 接口、静态网站等场景。

磁盘 IO 性能#

NVMe 固态硬盘是这个配置的一大亮点。首先看4K随机读写测试：

read: IOPS=3266, BW=12.8MiB/s
  clat (usec): avg=276.35

write: IOPS=3262, BW=12.7MiB/s
  clat (usec): avg=14.25

4K随机读写 IOPS 都在3200左右，带宽约13 MB/s，写入平均延迟仅14微秒。这个成绩相当优秀，说明确实使用了真实的 NVMe 固态硬盘，而不是普通 SATA SSD。对于数据库、日志写入等大量小文件读写操作，这个性能足够流畅。

再看顺序读写测试：

read: IOPS=545, BW=545MiB/s
write: IOPS=580, BW=580MiB/s

顺序读取速度 545 MB/s，写入速度 580 MB/s。这个速度已经达到中高端 NVMe 硬盘的水平，比传统机械硬盘快 10 倍以上。无论是备份恢复、大文件传输还是安装软件包，都能感受到明显的速度提升。

需要注意的是，这是测试到香港本地服务器的速度。实际使用中，访问国内或国外网站的速度会受到国际线路、高峰期拥堵等因素影响，通常无法达到这个满速状态。但从机房本身的网络出口来看，基础设施是合格的。

回国网络表现测试#

对于选择香港 VPS 的用户来说，最关心的就是访问国内网站的速度和稳定性。下面通过多个维度的测试，展示 JustHost 香港机房连接中国大陆的实际网络表现。

全国多节点延迟分布#

使用 itDog 平台从国内多个城市测试访问 VPS 的延迟情况：

从测试结果看，三大运营商的延迟表现差异明显。中国移动的延迟最低，上海移动节点仅 79 毫秒，整体平均 244 毫秒；电信次之，平均 324 毫秒；联通表现最差，平均352毫秒。西部地区如西藏、四川的延迟普遍较高，接近400毫秒。

这个延迟数据比预期要高一些。正常情况下，香港到国内主要城市的延迟应该在 30-80 毫秒之间。超过 300 毫秒的平均延迟说明可能存在路由绕行或者测试时段网络拥堵的情况。对于要求低延迟的应用场景，比如实时游戏就无法使用。

网络丢包率测试#

4.35% 的丢包率属于偏高水平。理想状态下丢包率应该控制在 1% 以内，超过 5% 就会明显影响 TCP 连接的稳定性和传输速度。这个丢包率可能导致网页加载时偶尔卡顿，SSH 连接偶尔掉线，文件传输速度不稳定等问题。

丢包的原因可能有多种：国际出口拥堵、运营商 QoS 限制、网络路由不稳定等。建议在不同时段多次测试，如果持续存在较高丢包，说明线路质量确实一般。

路由追踪分析#

通过路由追踪可以看出数据包经过的节点和跳数。香港 VPS 理想的路由应该是国内 → 广州/深圳出口 → 香港，但是实测发现从从国内出发，会先到美国，然后才会回到香港，导致延迟高达 300ms。

实际下载速度稳定在13.2 Mbps（约1.65 MB/s），远低于100M宽带的理论速度。从每秒的带宽变化可以看出波动较大，从几百KB到3MB不等。测试过程中出现了1595次重传，这和之前4.35%的丢包率相吻合。

常见问题解答#

Q1：JustHost 香港机房适合搭建什么类型的网站？#

+
适合访问量不大的个人博客、企业官网、API 接口等轻量级应用。1核1G 配置运行 WordPress、Typecho 等常见 CMS 系统没有问题，但如果网站流量较大或者需要运行复杂插件，建议升级到2核或更高配置。

Q2：为什么回国延迟这么高，比其他香港 VPS 慢很多？#

+

测试数据显示平均延迟303毫秒确实偏高。可能的原因包括路由绕行、测试时段网络拥堵、运营商 QoS 限制等。不同时段和不同运营商的表现会有差异，移动用户延迟相对较低。

如果对回国速度有较高要求，建议选择 CN2 GIA 线路的产品，例如 搬瓦工 。

Q3：JustHost 默认没有 Swap，如何添加交换空间？#

+

可以通过命令手动创建 Swap 文件。建议创建1GB 大小的 Swap 作为缓冲：

dd if=/dev/zero of=/swapfile bs=1M count=1024
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

然后编辑 /etc/fstab 添加 /swapfile none swap sw 0 0 使其开机自动挂载。

+
可以。测试显示能够解锁 Netflix 香港区、Disney+ 香港区、YouTube Premium 美国区等主流平台。但 Spotify 无法注册，TVBAnywhere+ 不可用。流媒体解锁情况可能随时变化，建议购买前先咨询客服确认。

Q5：25 端口能用吗？可以发邮件吗？#

+
